== Frequently asked questions ==

= How can I prevent fraudulent customers from completing transactions on the checkout page? =

To secure your checkout page against fraudulent users, simply activate the WooCommerce Fraud Prevention plugin. Then, enable the condition for the checkout page within the plugin settings. This allows you to block users by their email, IP address, state, or ZIP code, ensuring only genuine transactions proceed.

= What steps should I take to block fraudulent activity on the registration pages? =

Protect your registration pages by activating the WooCommerce Fraud Prevention plugin and enabling the specific condition for registration pages in the settings. This feature lets you block potential fraudsters based on their email, IP address, state, or ZIP code, safeguarding your site's integrity.

= Can the plugin block users based on their IP address? =

Yes, the WooCommerce Fraud Prevention plugin allows you to block users by IP address on both registration and checkout pages. You can enter multiple IP addresses into the plugin to cover various potential threats.

= Is it possible to block fraudsters by state, ZIP code, domain, IP address, and email address? =

Absolutely! Upon activating the plugin and configuring the necessary conditions for either registration or checkout pages, you can effectively block users based on a comprehensive list of identifiers, including state, ZIP code, domain, IP address, and email address.

= Can I customize error messages for users who are blocked? =

Yes, with the WooCommerce Fraud Prevention plugin, you have the flexibility to create custom error messages. This can be tailored specifically for each type of condition you've set, such as blocks by email, IP address, state, or ZIP code, enhancing the user experience even in denial situations.

= How can I utilize an external blacklist, like one from GitHub, with this plugin? =

The plugin supports the integration of external blacklists, including disposable/temporary email domains often listed on GitHub. Simply activate the plugin and enable the use of an external blacklist to enhance your site's protection against fraudulent entities.

= Where can I view a list of blocked users and their attempted transactions on my website? =

Once the WooCommerce Fraud Prevention plugin is activated, you'll have access to a comprehensive list of blocked users, including their email addresses and a record of how many times they attempted to place an order on your site. This feature gives you insightful data to monitor and adapt your security measures accordingly.

= Does the plugin offer detailed reports on fraudulent activities? =

Yes, the WooCommerce Fraud Prevention plugin provides detailed dashboard reports, presenting data on orders placed, users blocked, and suspected fraudulent activities, giving you clear insights into your store's security status.

= Can the plugin block users by additional parameters such as browser type? =

Indeed, beyond email and IP address, the plugin allows for advanced blocking based on a variety of parameters including browser type, domain, domain extension, and more, giving you a comprehensive toolset to protect your store.

= Is there a feature for importing or exporting blacklisted information? =

The plugin supports bulk upload functionalities for blacklisting, allowing you to upload an Excel sheet with blacklisted details such as email addresses, ZIP codes, or states. This streamlines managing your store's security efficiently.

= Can I create specific fraud identification rules? =

Customize your store's security by establishing your own fraud identification rules and risk thresholds within the plugin. This enables you to tailor the security measures to fit your specific business needs perfectly.

= Is there support for importing/exporting plugin settings for site migrations? =

For users planning to migrate their WooCommerce store, the plugin facilitates importing/exporting of settings via a JSON file. This ensures that you won't have to start from scratch, preserving your meticulously configured settings.
